Documentation

Vim's documentation is one of its greatest strengths, and is a great source for building on the Vim knowledge you learned in this screencast. The only drawback is it can be quite hard to find what you need if you don't know what to look for.

You can find an online version of Vim's documentation, but it's much easier to access it within Vim.

You can access the documentation with :help, or :h for short. It's divided into two manuals: the user manual, which explains how to accomplish various tasks, and the reference manual which goes into the nitty gritty.

You can go straight to a topic by searching for it. 

:h autocmd

If you're not quite sure what to look for, use auto-completion. Type :h with a few letters and then <Tab> to show the possibilities.

Every subject is tagged with one or more keywords in an alternate color. If your cursor is on a subject tag, you can jump to it with CTRL-]. To get back again, type CTRL-T or CTRL-O.

You can close the help window with :q.

So, for example, to look up what the gf command does, type :h gf. You'll see that it depends on the path option; to check on that quickly, type /path to go to the word, then hit CTRL-] to jump into it. When you've read enough about path, CTRL-T will take you back to where you were.

Finally, if you open up the main help with :h and scroll down past the reference manual, you'll come to a section called Local Additions. This links up the documentation for all the plugins you have installed (but only if you've read in their help docs with :helptags ~/.vim/doc or :call pathogen#helptags()).

I recommend you dip into the documentation as often as you can. It's a great resource and you'll always learn something new.
