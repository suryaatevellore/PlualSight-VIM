Philosophy

Vim looks complex but is built on only two simple ideas. Once you've grasped them, everything else will fall into place. These two ideas are also the ones that distinguish Vim from most other text editors.

The first idea is modal editing and the second is operators.

Modal editing

Vim is all about editing text efficiently. Have you ever thought about how you write code, or any text for that matter? You spend more time moving through existing text and editing it than you do typing fresh blocks of the stuff. So it's sensible to make navigation and editing as easy as, or even easier than, typing it the first time. And this is what Vim does.

Most software prioritises text input over text editing: when your cursor is in the right place you just start typing. Every word processor works this way. If you want to cut and paste, or search, or change a word, you have to pull down a menu or use some combination of modifier keys.

Vim turns this on its head: it knows that you spend more time editing than performing data entry, so it makes editing and navigation just as easy as, or perhaps slightly easier than, typing text from scratch. As an example, if you want to move down a line or jump to the end of the document, you do it with single keys like j and G.

And again, in the pursuit of efficiency, the things you do most often are on the home row of the QWERTY keyboard. There's rarely a need to move your hands away from the touch-typing position to move the cursor.

So how does Vim know what you mean when you press j? Should it take you down one line, or type the letter j in the text? It depends on the mode. Pressing j in Command mode moves the cursor down a line. Pressing the same joy in Input mode adds a j to the text.

This may sound confusing, especially when you hear there are five main modes -- Normal, Insert, Visual, Replace, and Command-line -- and a few others, too. If keys behaves differently in each mode, how is anyone supposed to remember all the combinations?

The big insight here is...you can ignore all the modes. Don't worry about them. Think about this instead: you simply give Vim one command after another. For example: move down three lines, go to the opening bracket, insert some text, find the next opening bracket, and so on. That's it.

Some commands are self contained, such as 'delete the character under the cursor'; Vim knows what you want to do as soon as it sees the command. For other commands, you need to tell Vim when you're ready to go. For example, if you insert some text, Vim doesn't know whether you're intending to type all 1,200 pages of War and Peace or just write "Hello World"; you have to tell it when you've finished.

So when you're getting used to Vim, think to yourself 'command ... command ... command' and you'll be fine. You know this is what Vim wants because it starts up in Command mode. And indeed Command mode is more commonly known as "Normal" mode.

You may be thinking, "This is all very efficient, but how do I actually type in my code?" It's a single command: i. This letter tells Vim you want to insert some text. When you're done, hit <escape> to let Vim know you've finished.

iIt was a dark and stormy night<escape>

You'll find you use a lot because it always takes you back to Normal mode, ready for the next command.

Operators

Vim has many motion commands, which we'll cover in depth later. If you currently move your cursor with the arrow keys, <page up>/<page down> or <home>/<end>, you are in for a treat. With Vim you'll be able to move your cursor quickly and with far more precision.

But even better, you can change the text while you're moving over it. This is a powerful workflow that you'll use all the time.

Here are two examples:

  • To delete from the cursor to the end of the line, type d$. d for delete, and $ to continue deleting all the way to the end of the line.

  • To change all the code from the cursor to the second occurrence of end, type this:

    c2/end<Return>your new code<Escape>

c for change, 2 for the second match, /end<Return> to search for the word end. Type your new code, then hit Escape to resume Command mode.

It may look cryptic, but you'll find it makes more sense when you use it.

In these examples the d and c are operator commands. When an operator command is followed by a motion command, it tells Vim you want to operate on the text while you move over it.

Operators can delete or change text, copy it, format it, or change its case, indent or outdent it.

They'll feel more natural as you get used to the various motion commands.

Help

Which brings us briefly to Vim's extensive help system. To get help on any command or topic, use "colon h" and a term to search for.

:h movement
:h j

To exit, type "colon bd", for "buffer delete."

:bd

Lower Third: :q also closes help, but will also quit the entire application in some cases

Vim's help system is extremely useful when you're starting out, and is even useful for experts.

Conventions and mnemonics

In Vim, the keyboard is king; you won't find menu items or toolbar buttons for operators. That can be a lot of keys to remember, so Vim tries to help you.

The first convention is that upper case commands are usually "super-sized" versions of their lower case counterparts. Lower case i inserts text where the cursor is; capital I inserts text at the start of the line.

Lower Third: Use 'j' or arrow keys to move downward

Lower case w moves the cursor forward one word; capital W treats contiguous code as one word and jumps forward to the next meaningful code "word."

items.each { |item| p item }
 ^

items.each { |item| p item }
     ^     ^
     w     W

Lower Third: 'b' moves backward by words, also "B"
Sidebar:

      B b --- w W

Another convention is the forceful exclamation mark. Adding an exclamation mark to the end of a command will force it to happen without confirmation. To quit Vim, type :q. But Vim won't quit if you have unsaved changes; force it to quit without confirmation by adding an exclamation mark. 

:q!

As always with Vim, it's all about efficiency. The most commonly used commands are placed on the home keys and have the best mnemonics. By the time you get to less frequently used commands, all the good mnemonics have been taken so their keys become slightly harder to remember.

For example, moving the cursor is a common task, so these are under the strong fingers of the right hand. You'll delete text frequently, so the sensible d deletes.

 h                j              k            l
(Back one letter)(Down one line)(up one line)(Forward one letter)

Slightly less obviously, the code-folding commands all begin with z because it looks like a folded sheet of paper.

That's enough theory. There's no substitute for using Vim, so let's install it and put it to work!
