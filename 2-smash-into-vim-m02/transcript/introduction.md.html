Introduction

It's PeepCode!

In the next hour, you will become familiar with the Vim text editor.

Vim was specifically built to help programmers edit text more efficiently. Because of its origins in the Terminal, it can be daunting at first. But with a little practice, you will find it well worth the effort. I think you'll find that it's not as hard as it looks: once you've grasped two main concepts it will all make sense.

Learning the piano is also said to be worthwhile, but only if you want to play music. How do you decide whether learning Vim will pay off for you? Here are some areas where Vim shines:

  • It's on every platform.
  • It's entirely keyboard driven (though you can use a mouse if you want).
  • It's precise: you'll feel like a text surgeon.
  • It's powerful: you can make sweeping changes with just a few keystrokes. Unlike some other editors, those keystrokes won't strain your fingers.
  • It works very well over SSH, even from an iPhone!.
  • It's highly configurable.
  • You can extend it with plugins to do exactly what you want.
  • If you use MacVim on Mac OS X, many standard keyboard shortcuts will work as you expect them to, right out of the box.
  • The documentation is excellent.
  • Finally, you'll build skills that you can use for years, or maybe even decades!

Lighthearted feuds between users of various text editors are well known, but we sincerely want you to use the editor that you find most comfortable. Here are some areas where Vim is suboptimal:

  • Many of its defaults are not what you want. Fortunately that's easy to fix.
  • Its scripting language is not a full-blown language like Ruby or Lisp, but the clunky Vimscript. You can work around this to an extent with the Ruby interface, but it does put an upper bound on the sophistication of things like indentation logic.
  • It is sometimes too easy to perform destructive tasks such as overwriting the paste buffer.
  • Its default regular expression syntax is inconsistent.

Of course, there are other great text editors out there, including TextMate and Emacs. The writers and editors of this screencast have done time with all three, so we'd like to think we could go a full 12 rounds for or against any of them. It doesn't really matter which one you choose, but you'll benefit most if you pick one and master it. If you're currently undecided, this screencast will help you figure out whether Vim is the best one for you. Or, you might steal some of Vim's best ideas back to your editor of choice.

TextMate devotees will benefit too: when it's time to edit your Linux server's settings -- Apache configs perhaps -- you'll have the knowledge and confidence to fire up Vim and get it done stat so you can return straightaway to the comfort of that gorgeous TextMate GUI.

History

Vim started life in 1991 as Bram Moolenaar's improvement on Bill Joy's venerable Vi editor, which itself began in 1976. The name vim comes from Vi IMproved. Bram still leads Vim's development today, almost 20 years later. Few open source projects have been actively developed for this long or with this continuity, and it shows in Vim's stability, speed, documentation, and community.

Vim is charityware: it comes for free but you are asked to consider donating to the Kibaale Children's Centre in Uganda. You can find more details on Vim's website.

Learning Vim
