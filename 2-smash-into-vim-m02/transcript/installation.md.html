Installation

Vim is available on Unix, Mac OS X, Windows, and other operating systems. Refer to the Vim website for a full list of downloads.

When producing this screencast, the latest stable version of Vim was 7.2.

:version

Mac OS X

There is an excellent port of Vim 7.2 for Mac OS X, called MacVim. It is available as a universal binary.

We recommend getting the latest snapshot release because the latest stable release is more than a year older.

Be careful...there are two different ports called MacVim. The best one is at Google code, not MacVim.org.

Download the application, uncompress it, and copy it to your "Applications" directory.

You can also build it from source if you want to. There are good instructions on the website.

Once you have installed MacVim, give your shell the good news: copy the bundled mvim shell script to a directory in your path. We also recommend that you symlink vim to it as a shortcut if you want to run the latest vim in the Terminal. So if ~/bin is in your path, copy and link it with these commands:

$ cp ~/Downloads/MacVim-snapshot-51/mvim ~/bin/
$ ln -nfs ~/bin/mvim ~/bin/vim

Then in your Terminal you can fire up MacVim with mvim <filename> or the GUI-less Vim with vim <filename>.

Finally, spread the love to tools like crontab and git rebase -i by setting up your shell's EDITOR variable (probably in the .bashrc file in your home directory):

export EDITOR='/Users/peepcode/bin/vim'

Be sure to point EDITOR to vim, not mvim.

Unix / Linux

Vim is probably already available on your system but it might not be the latest version. Or, it might not have been compiled with the features you want.

Compiling Vim from source is both easy and the best way to go. Download the latest version and unzip it.

$ wget ftp.vim.org/pub/vim/unix/vim-7.2.tar.bz2
$ bunzip2 -c vim-7.2.tar.bz2 | tar -xf -
$ cd vim72

Vim has many optional features which can be compiled in. To make things easier, they are grouped into feature lists: tiny, small, normal (the default), big, and huge. Unless you are running on a puny system I recommend treating yourself to the huge list of features. Do this in the configure step:

$ ./configure --with-features=huge

You can also compile in Ruby support (and/or Perl, Python and Tcl). This doesn't affect whether or not you can edit Ruby files in Vim; instead it allows you to run Ruby commands within Vim, and to script Vim with Ruby. To do this, use a configure step like this:

$ ./configure --with-features=huge --enable-rubyinterp

By default Vim will install into /usr/local. To install elsewhere, specify it with the standard --prefix flag.

$ ./configure --with-features=huge --enable-rubyinterp --prefix=/your/path/here

Now you can build Vim with the standard make and make install:

$ make
$ make install

Experts can fine-tune which features are compiled in by editing the Makefile instead of specifying a feature list. You can see the full list with the --help flag:

$ ./configure --help

To verify success, type:

$ vim --version

You should see the version number and a long list of enabled or disabled features.

Windows

On Windows the best option is to install a pre-built binary for GVim, the graphical version of Vim. Go to Vim's downloads page and get the self-installing executable, named gvim72.exe or similar. Double-click the file to start the installer; it's a good idea to tick the box labelled Create .bat files for command line use.

If you use a 64 bit version of Windows, you could follow the instructions at the Google Code project.

Cygwin users can get hold of Vim through Cygwin's installer.

Once you have installed gVim, use gvim rather than vim on the command line.
